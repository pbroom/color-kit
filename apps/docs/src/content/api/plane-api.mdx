# Plane API

The Plane API is a geometry-first layer for color tooling. It lets you:

- define a normalized 2D plane in multiple color models
- run deterministic geometry queries
- compile points and paths into SVG-ready output
- reuse query results with stable cache keys

It is intentionally UI-agnostic and pairs with `color-kit/react` adapters.

## definePlane()

`definePlane()` maps a plane onto a model space we can work with. AI really wants me to use the word "normalized" here to indicate the plane works with whatever color model is selected.

```ts
import { definePlane } from 'color-kit';

const plane = definePlane();

// Equivalent to:
const plane = definePlane({
  model: 'oklch',
  x: { channel: 'l', range: [0, 1] },
  y: { channel: 'c', range: [0.4, 0] },
  fixed: { l: 0.5, c: 0, h: 0, alpha: 1 },
});
```

When you need to preserve intent across model switches, prefer anchoring the
plane to a real color instead of manually carrying model-relative fixed values:

```ts
import { definePlane, definePlaneFromColor, parse } from 'color-kit';

const color = parse('#7c3aed');

const hslPlane = definePlane({
  model: 'hsl',
  color,
  x: { channel: 'h' },
  y: { channel: 's' },
});

const rgbPlane = definePlaneFromColor(color, {
  model: 'rgb',
  x: { channel: 'r' },
  y: { channel: 'g' },
  fixed: { b: 64 },
});
```

`color` fills in model-relative defaults from the selected model, and explicit
`fixed` values still win. This is especially useful in UIs where the user can
switch between `oklch`, `hsl`, `rgb`, or `p3` without rebuilding every fixed
channel bag by hand.

If you use TypeScript, `definePlane()` now narrows `x`, `y`, and `fixed` to the
selected `model`, so invalid channel/model combinations are caught before
runtime.

### `sense()`

Use `sense()` when you want a fluent query API bound to one plane definition. It is the ergonomic option for running multiple queries against the same plane.

```ts
import { definePlane, sense } from 'color-kit';

const plane = definePlane({
  fixed: { h: 280 },
});

const query = sense(plane);

const boundary = query.gamutBoundary({ gamut: 'display-p3' });
const band = query.chromaBand({ requestedChroma: 0.22 });
```

`gamutRegion()` is the multi-path complement to `gamutBoundary()`: it returns
`boundaryPaths` for the visible contour fragments and a `visibleRegion` payload
you can pass to `toSvgCompoundPath()`. By default it is viewport-first, and
`scope: 'full'` opts into offscreen boundary geometry when you need an export or
overview pass.

### `toSvgPath()`

Use `toSvgPath()` to compile an ordered list of normalized plane points into an SVG `d` string. It is the last step after running a query that returns a single point sequence.

```ts
import { definePlane, sense, toSvgPath } from 'color-kit';

const plane = definePlane({
  fixed: { h: 280 },
});

const boundary = sense(plane).gamutBoundary({ gamut: 'srgb' });

const d = toSvgPath(boundary.points, {
  closeLoop: true,
  scale: 100,
  precision: 3,
});
```

## Model Support

`definePlane({ model })` supports these models:

- `oklch` (`l`, `c`, `h`)
- `rgb` (`r`, `g`, `b`)
- `hsl` (`h`, `s`, `l`)
- `hsv` (`h`, `s`, `v`)
- `oklab` (`L`, `a`, `b`)
- `hct` (`h`, `c`, `t`)
- `p3` (Display P3; `r`, `g`, `b`)
- `display-p3` (legacy alias for `p3`)

`definePlane({ model: 'p3' })` and `definePlane({ model: 'display-p3' })` both
select the Display P3 plane model. Resolved planes canonicalize that alias back
to `p3`, while query options such as `gamutBoundary({ gamut: 'display-p3' })`
still use the full `display-p3` gamut token.

Internally, query computation remains canonical in OKLCH. Mapping helpers (`colorToPlane`, `planeToColor`, `projectRegionBetweenPlanes`, `fallbackPoint`, `gradient`) convert through model adapters, while LC-specific geometry queries remain capability-gated.

## Plane Definition and Mapping

Use `definePlane()` to normalize ranges and fixed channels. All inputs are optional. Defaults are model-specific:

- axis channels use model defaults (for example `oklch` defaults to `x: l`, `y: c`)
- ranges use model defaults (for `oklch`, `PLANE_DEFAULT_RANGES`)
- `fixed` channels are interpreted in the selected model

```ts
import { colorToPlane, definePlane, planeToColor } from 'color-kit';

const resolvedPlane = definePlane({
  x: { channel: 'l', range: [0.1, 0.95] },
  y: { channel: 'c', range: [0, 0.32] },
  fixed: { h: 280, alpha: 1 },
});

const point = colorToPlane(resolvedPlane, { l: 0.6, c: 0.2, h: 280, alpha: 1 });
const color = planeToColor(resolvedPlane, point);
```

## Query Execution Patterns

Use either the fluent API (`sense`) or stateless helpers (`runPlaneQuery` / `runPlaneQueries`).

```ts
import { runPlaneQueries, sense } from 'color-kit';

const query = sense(plane);
const boundary = query.gamutBoundary({ gamut: 'srgb' });

const results = runPlaneQueries(plane, [
  { kind: 'chromaBand', requestedChroma: 0.22, gamut: 'display-p3' },
  {
    kind: 'fallbackPoint',
    color: { l: 0.7, c: 0.28, h: 280, alpha: 1 },
    gamut: 'srgb',
  },
]);
```

For per-query details, see [Plane Queries](/docs/api/plane-queries).

## Compile and Cache

Use compile helpers to convert points/paths to deterministic SVG `d` strings.

```ts
import {
  PlaneQueryCache,
  runCachedPlaneQuery,
  runPlaneQuery,
  sense,
  toSvgCompoundPath,
  toSvgPath,
} from 'color-kit';

const cache = new PlaneQueryCache();
const boundaryQuery = {
  kind: 'gamutBoundary',
  gamut: 'display-p3',
  samplingMode: 'adaptive',
} as const;

const boundary = runCachedPlaneQuery(cache, plane, boundaryQuery, () =>
  runPlaneQuery(plane, boundaryQuery),
);
const query = sense(plane);
const region = query.contrastRegion({
  reference: { l: 1, c: 0, h: 0, alpha: 1 },
  threshold: 4.5,
});

const boundaryD = toSvgPath(boundary.points, { closeLoop: true });
const regionD = toSvgCompoundPath(region.paths, { closeLoop: true });
```

## Region Geometry Operations

Region utilities are useful when you need set operations or post-processing.

```ts
import { parse } from 'color-kit';
import {
  differenceRegions,
  rotateRegion,
  scaleRegion,
  unionRegions,
} from 'color-kit';

const reference = parse('#ffffff');
const aa = runPlaneQuery(plane, {
  kind: 'contrastRegion',
  reference,
  threshold: 4.5,
});
const aaa = runPlaneQuery(plane, {
  kind: 'contrastRegion',
  reference,
  threshold: 7,
});

const aaOnly = differenceRegions(aa, aaa, { resolution: 96 });
const merged = unionRegions(aaOnly, aaa, { resolution: 96 });
const transformed = rotateRegion(scaleRegion(merged, 0.96), 6);
```

## React Bridge (`ColorApi`)

`color-kit/react` exposes adapter helpers that map Plane API results into ColorArea UI coordinates.

```ts
import { ColorApi } from 'color-kit/react';

const axes = ColorApi.resolveColorAreaAxes({
  x: { channel: 'l' },
  y: { channel: 'c' },
});

const boundary = ColorApi.getColorAreaGamutBoundaryPoints(
  referenceColor.h,
  axes,
  {
    gamut: 'display-p3',
    samplingMode: 'adaptive',
  },
);
```

## API Surface (Grouped)

- **Definition + mapping**
  - `definePlane`, `definePlaneFromColor`, `planeHue`, `colorToPlane`, `planeToColor`
- **Queries**
  - `sense`, `runPlaneQuery`, `runPlaneQueries`, `colorAtPlanePoint`, `gamutRegion`
- **Compile + cache**
  - `toSvgPath`, `toSvgCompoundPath`, `createPlaneQueryKey`, `PlaneQueryCache`, `runCachedPlaneQuery`
- **Geometry operations**
  - `unionRegions`, `intersectRegions`, `differenceRegions`, `containsPoint`, `nearestPointOnPath`, `pointDistance`
- **Transforms**
  - `translateRegion`, `scaleRegion`, `rotateRegion`, `projectRegionBetweenPlanes`
- **React adapters**
  - `ColorApi.getColorAreaGamutBoundaryPoints`, `ColorApi.getColorAreaContrastRegionPaths`, `ColorApi.getColorAreaChromaBandPoints`, `ColorApi.getColorAreaFallbackPoint`
