# Plane Queries

<p className="docs-lede">
  Deterministic geometry queries for a resolved plane. Each query returns typed
  results with pre-computed points and paths ready for rendering.
</p>

<div className="prose-api">

## Creating a Query

Use the fluent API to create a query object from a resolved plane:

```ts
const query = sense(plane);
```

Or execute stateless query objects directly:

```ts
runPlaneQuery(plane, { kind: 'gamutBoundary', gamut: 'srgb' });
```

## Shared Notes

- `samplingMode` controls contour extraction strategy:
  - `hybrid` (default for contrast queries) &mdash; direct root solving + adaptive tracing for smoother boundaries.
  - `uniform` &mdash; fixed stepping across the plane (legacy compatibility).
  - `adaptive` &mdash; recursive marching refinement (legacy compatibility).
- `gamutRegion()` is viewport-first by default:
  - `scope: 'viewport'` returns only boundary fragments that intersect the current plane window.
  - `scope: 'full'` opts into offscreen boundary geometry for exports or overview visualizations.
- `metric` is available for `contrastBoundary` / `contrastRegion`:
  - `wcag` &mdash; threshold uses ratio values such as `3`, `4.5`, `7`.
  - `apca` &mdash; threshold uses normalized `Lc` values (for example `0.3`, `0.45`, `0.6`) and supports `apcaPolarity` + `apcaRole`.
- On planes that are not `oklch` with `l/c` (or `c/l`) axes, LC-specific queries intentionally return empty geometry.

## Query Capability Matrix

| Query              | Multi-model mapping support        | Requires `oklch` + `l/c` axes            |
| ------------------ | ---------------------------------- | ---------------------------------------- |
| `gamutBoundary`    | Returns typed result for any model | Yes (`points` is empty when unsupported) |
| `gamutRegion`      | Yes                                | No                                       |
| `contrastBoundary` | Returns typed result for any model | Yes (`points` is empty when unsupported) |
| `contrastRegion`   | Returns typed result for any model | Yes (`paths` is empty when unsupported)  |
| `chromaBand`       | Returns typed result for any model | Yes (`points` is empty when unsupported) |
| `fallbackPoint`    | Yes                                | No                                       |
| `gradient`         | Yes                                | No                                       |

## `gamutBoundary`

Returns the boundary contour of a color gamut projected onto the plane.

```ts
const boundary = query.gamutBoundary({
  gamut: 'srgb',
  hue: 220,
  samplingMode: 'adaptive',
  simplifyTolerance: 0.001,
});
```

  Returns <code>PlaneGamutBoundaryResult</code>
</p>

  </li>
</ul>

## `gamutRegion`

Returns the visible in-gamut region for the current plane window plus any
boundary fragments that cross that viewport.

```ts
const gamut = query.gamutRegion({
  gamut: 'srgb',
  scope: 'viewport',
});
```

<p className="api-returns">
  Returns <code>PlaneGamutRegionResult</code>
</p>

<ul className="api-field-list">
  <li>
    <code>kind</code> &mdash; <code>'gamutRegion'</code>
  </li>
  <li>
    <code>gamut</code> &mdash; <code>'srgb' | 'display-p3'</code>
  </li>
  <li>
    <code>scope</code> &mdash; <code>'viewport' | 'full'</code>
  </li>
  <li>
    <code>viewportRelation</code> &mdash;{' '}
    <code>'inside' | 'outside' | 'intersects'</code>
  </li>
  <li>
    <code>solver</code> &mdash; reports whether the result came from a domain
    clip, an analytic shortcut, or an implicit contour pass
  </li>
  <li>
    <code>boundaryPaths</code> &mdash; <code>{'PlanePoint[][]'}</code> for the
    visible contour fragments (or full projected contour when
    <code>scope: 'full'</code>)
  </li>
  <li>
    <code>visibleRegion</code> &mdash;{' '}
    <code>{'{ paths: PlanePoint[][] }'}</code> clipped to the current viewport
  </li>
</ul>

## `contrastBoundary`

Returns a contour where colors on the plane hit a given contrast threshold against a reference color.

```ts
const line = query.contrastBoundary({
  reference: parse('#ffffff'),
  metric: 'apca',
  threshold: 0.45,
  apcaPolarity: 'absolute',
  apcaRole: 'sample-text',
  gamut: 'display-p3',
  samplingMode: 'hybrid',
});
```

<p className="api-returns">
  Returns <code>PlaneContrastBoundaryResult</code> with one contour in{' '}
  <code>points</code>.
</p>

## `contrastRegion`

Returns filled region paths where all colors meet a contrast level against the reference.

```ts
const region = query.contrastRegion({
  reference: parse('#111827'),
  metric: 'wcag',
  level: 'AA',
  gamut: 'srgb',
  samplingMode: 'hybrid',
});
```

<p className="api-returns">
  Returns <code>PlaneContrastRegionResult</code> with{' '}
  <code>{'paths: PlaneRegionPoint[][]'}</code>.
</p>

## `chromaBand`

Samples a band of colors at a given chroma level, useful for showing achievable lightness range.

```ts
const band = query.chromaBand({
  mode: 'proportional',
  requestedChroma: 0.2,
  selectedLightness: 0.56,
  gamut: 'display-p3',
});
```

<p className="api-returns">
  Returns <code>PlaneChromaBandResult</code> with sampled <code>points</code>.
</p>

## `fallbackPoint`

Finds the closest in-gamut point for a color that falls outside the target gamut on the plane.

```ts
const fallback = query.fallbackPoint({
  color: requested,
  gamut: 'srgb',
});
```

<p className="api-returns">
  Returns <code>PlaneFallbackPointResult</code>
</p>

<ul className="api-field-list">
  <li>
    <code>gamut</code> &mdash; the target gamut
  </li>
  <li>
    <code>point</code> &mdash; <code>{'{ x, y, color }'}</code> where{' '}
    <code>color</code> is mapped into the target gamut
  </li>
</ul>

## `gradient`

Computes evenly-spaced points along a gradient path between two colors on the plane.

```ts
const gradient = query.gradient({
  from: parse('#3b82f6'),
  to: parse('#ef4444'),
  steps: 32,
});
```

<p className="api-returns">
  Returns <code>PlaneGradientResult</code> with{' '}
  <code>{'points: Array<{ x, y, color }>'}</code>.
</p>

## Stateless Query Objects

`runPlaneQuery` and `runPlaneQueries` accept plain objects instead of fluent method calls. This is useful for queueing work, caching, and passing queries across worker boundaries.

```ts
import { runPlaneQueries } from 'color-kit';

const [boundary, fallback] = runPlaneQueries(plane, [
  { kind: 'gamutBoundary', gamut: 'display-p3', samplingMode: 'adaptive' },
  { kind: 'fallbackPoint', color: requested, gamut: 'srgb' },
]);
```
