# React Primitives

Color Kit React primitives are intentionally headless building blocks for design-tool workflows.

## What makes them "primitives"

- They own behavior and semantics, not visual branding.
- They expose deterministic requested/displayed contracts.
- They remain composable in controlled and uncontrolled setups.

## Typical composition

```tsx
import {
  Color,
  ColorArea,
  ColorSlider,
  ColorInput,
  ColorStringInput,
} from '@color-kit/react';

export function Picker() {
  return (
    <Color defaultColor="#3b82f6">
      <ColorArea />
      <ColorSlider channel="h" />
      <ColorSlider channel="alpha" />
      <ColorInput model="oklch" channel="h" />
      <ColorStringInput format="oklch" />
    </Color>
  );
}
```

## API design goals

- Stable, composable contracts.
- Intent preservation under gamut constraints.
- Accessibility-first interaction defaults.
